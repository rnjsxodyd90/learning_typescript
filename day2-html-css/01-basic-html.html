Interview Tip: Headings are important for SEO (Search Engine Optimization) and accessibility. Google reads your headings to understand your page content. Screen readers use headings to help blind users navigate. Always use headings in order (don't skip from h1 to h4).

Lesson 2: Paragraphs and Text Formatting

This is a paragraph. It's the most basic way to put text on a page. Each paragraph gets its own block of space. Notice how there's a gap between this paragraph and the next one? That's automatic.

This is a second paragraph. See the space between them? You didn't have to add that -- the browser does it automatically because each <p> is a separate block.

You can make text bold using the strong tag. You can make text italic using the em tag. You can even combine bold and italic.

The <br> tag creates a line break.
See? This text is on a new line, but still in the same paragraph.
It's like pressing Enter once (not twice).

Why <strong> instead of <b>? Both make text bold, but <strong> also tells screen readers "this text is important." Similarly, <em> is preferred over <i> for italics. In interviews, use the "semantic" (meaningful) tags: strong and em.

  1. Preheat the oven to 350 degrees
  2. Mix the ingredients
  3. Pour into a pan
  4. Bake for 30 minutes

Nested Lists (a list inside a list):

  • Fruits
    • Apples
    • Bananas
  • Vegetables
    • Carrots
    • Broccoli
Common Mistake: Don't put content directly inside <ul> or <ol>. Always wrap each item in <li> tags.
Wrong: <ul>Apple</ul>
Right: <ul><li>Apple</li></ul>

Lesson 6: Divs and Spans (Containers)

Everything in this box is grouped together by the div.

This is a separate group from the blue one above.

In this sentence, these words are red and bold because they're wrapped in a span with styling. The rest of the text is normal.

DIV vs SPAN - The Simple Rule:
<div> = BOX (takes full width, starts new line) -- use for grouping sections
<span> = HIGHLIGHTER (inline, doesn't break the line) -- use for styling words within text

In Angular, you'll use <div> constantly for layout.

Bonus: Semantic HTML (Know This for Interviews)

Instead of using <div> for everything, use meaningful tags:

  • <header> - The top section of a page (logo, navigation)
  • <nav> - Navigation links (menus)
  • <main> - The main content of the page
  • <section> - A thematic section of content
  • <article> - Self-contained content (like a blog post)
  • <aside> - Side content (like a sidebar)
  • <footer> - The bottom section of a page (copyright, links)
Interview Tip: If an interviewer asks "What is semantic HTML?" say: "Semantic HTML uses tags that describe the meaning of content, like <header>, <nav>, and <footer>, instead of generic <div> tags. It improves accessibility, SEO, and code readability."

How Responsive Design Works

This page uses media queries to change the layout at different screen widths:

DESKTOP Above 768px -- cards in a row, 2-column layout
TABLET 577px - 768px -- 2-column becomes stacked
PHONE Below 576px -- everything stacks, navbar vertical
How to test:
1. DRAG the edge of your browser window to make it narrower/wider
2. OR press F12 to open Developer Tools, then click the phone/tablet icon (top-left of the DevTools panel) to simulate devices

Media Query Cheat Sheet

/* Default styles go here (apply to all sizes) */
.my-element {
  font-size: 16px;
  display: flex;
  flex-direction: row;   /* side by side on desktop */
}

/* Tablet: 768px or narrower */
@media (max-width: 768px) {
  .my-element {
    flex-direction: column;   /* stack on tablet */
    font-size: 14px;
  }
}

/* Phone: 576px or narrower */
@media (max-width: 576px) {
  .my-element {
    padding: 10px;   /* less padding on phone */
    font-size: 12px;
  }
}

Common responsive patterns:

  • Change flex-direction: row to column (side-by-side becomes stacked)
  • Change width: 33% to width: 100% (3 columns becomes 1)
  • Reduce font-size and padding on smaller screens
  • Hide non-essential elements with display: none
  • Replace horizontal nav with a hamburger menu
Interview Tip: If asked "How do you make a website responsive?" answer: "I use a mobile-first approach with CSS media queries to adjust layout, font sizes, and spacing at different breakpoints. I use Flexbox with flex-wrap for fluid card layouts, and change flex-direction from row to column for stacked mobile layouts."

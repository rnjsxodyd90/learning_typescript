Lesson 2: HTML Forms

Forms are how users INPUT data on a website. Every login page, every sign-up page, every search bar -- they're all forms. Let's learn every form element you need to know.

The <form> Tag

Every form starts with the <form> tag:

<form action="/submit" method="POST">
  ... your inputs go here ...
</form>
For now: We won't use action or method in our examples because we have no server to send data to. In Angular, form submission is handled differently (in TypeScript).

Text Inputs

The "name" attribute: When a form is submitted, the data is sent as key-value pairs. The name attribute is the KEY.
So if you type "Sura" in the name field, the server receives: username=Sura

Textarea (Multi-line Text)

Input vs Textarea:
<input type="text"> = single line (name, email, search)
<textarea> = multiple lines (comments, descriptions, messages)

Checkbox vs Radio -- THE KEY DIFFERENCE:
Checkbox = pick MANY (square boxes). Each is independent.
Radio = pick ONE (circles). They share the same name attribute.

This WILL come up in interviews. Remember: checkbox = many, radio = one.

Dropdown Menus (Select)

When to use what:
2-5 options? Use radio buttons (user sees all options at once)
5+ options? Use a dropdown (saves space on the page)

Buttons: <button> vs <input type="submit">

Option 1: The <button> tag (PREFERRED)

Option 2: The <input type="submit"> tag (older style)

Other button types:

IMPORTANT for Angular: In Angular, you'll almost always use <button type="button"> or <button type="submit"> and handle the action in TypeScript. You'll rarely use <input type="submit">.
